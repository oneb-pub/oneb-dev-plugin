# Шаблон tech_debt/index.md

Коли каталог `tech_debt/` створюється з нуля, скопіюй цей скелет у
`tech_debt/index.md` і додай перший рядок.

## Порожній скелет

```markdown
# Tech Debt — реєстр

Реєстр відкритих технічних боргів. Закриті борги фізично видаляються звідси
та з `items/` — історія залишається в git.

Як читати таблицю:
- **#** — трицифровий номер, відповідає файлу `items/NNN-*.md`
- **Пріоритет**: 🔴 високий (блокує / баги) · 🟡 середній (DX/UX) · 🟢 низький (приємно мати)
- **Статус**: `відкрито` для всіх рядків (закриті видаляються)

| #   | Назва                                          | Категорія | Пріоритет   | Статус   |
| --- | ---------------------------------------------- | --------- | ----------- | -------- |
```

## Приклад заповненої таблиці

```markdown
| #   | Назва                                          | Категорія              | Пріоритет   | Статус   |
| --- | ---------------------------------------------- | ---------------------- | ----------- | -------- |
| 001 | [Винести магічні константи](items/001-magic-constants.md) | Backend / Domain model | 🟡 середній | відкрито |
| 002 | [Налаштувати Sentry](items/002-sentry.md)      | Observability          | 🔴 високий  | відкрито |
| 003 | [Покрити тестами PaymentService](items/003-payment-tests.md) | Testing | 🟢 низький  | відкрито |
```

## Правила оновлення

- При додаванні нового боргу: новий рядок у кінець, номер = попередній + 1.
- При закритті боргу: фізично видалити рядок (не позначати галочкою/закресленням).
- Колонки розділяти `|`, заголовок з `---` під ним — стандартний markdown-table.
- Назва завжди як markdown-посилання `[Текст](items/NNN-slug.md)`.
