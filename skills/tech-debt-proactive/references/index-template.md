# Шаблон tech_debt/index.md

Коли каталог `tech_debt/` створюється з нуля, скопіюй цей скелет у
`tech_debt/index.md` і додай перший рядок.

## Порожній скелет

```markdown
# Tech Debt — реєстр

Реєстр відкритих і відкладених технічних боргів. Виконані (закриті) та
видалені як неактуальні борги фізично видаляються звідси та з `items/` —
історія залишається в git.

Як читати таблицю:
- **#** — трицифровий номер, відповідає файлу `items/NNN-*.md`. Номери не
  переприсвоюються: якщо борг #005 закрито, наступний новий борг отримує
  номер (макс + 1), а не 005.
- **Пріоритет**: 🔴 високий (блокує / баги) · 🟡 середній (DX/UX) · 🟢 низький (приємно мати)
- **Статус**: `відкрито` (готовий до взяття у роботу) · `🕒 відкладено`
  (чекає на умову активації; в item-файлі вказано `Активувати після:`)

| #   | Назва                                          | Категорія | Пріоритет   | Статус       |
| --- | ---------------------------------------------- | --------- | ----------- | ------------ |
```

## Приклад заповненої таблиці

```markdown
| #   | Назва                                          | Категорія              | Пріоритет   | Статус        |
| --- | ---------------------------------------------- | ---------------------- | ----------- | ------------- |
| 001 | [Винести магічні константи](items/001-magic-constants.md) | Backend / Domain model | 🟡 середній | відкрито      |
| 002 | [Налаштувати Sentry](items/002-sentry.md)      | Observability          | 🔴 високий  | відкрито      |
| 003 | [Покрити тестами PaymentService](items/003-payment-tests.md) | Testing                | 🟢 низький  | відкрито      |
| 014 | [Перейти на GraphQL](items/014-graphql.md)     | Backend / API          | 🟢 низький  | 🕒 відкладено |
```

## Правила оновлення

- **При додаванні нового боргу:** новий рядок у кінець, номер = `(макс у
  таблиці) + 1`. Не використовувати «дірки», що залишились від видалених
  чи закритих боргів.
- **При виконанні боргу:** фізично видалити рядок (не позначати галочкою
  чи закресленням). Item-файл також видаляється. Решта рядків залишається
  без перенумерації.
- **При видаленні боргу як неактуального:** так само — фізично видалити
  рядок і item-файл. У повідомленні коміту явно вказати причину
  видалення, щоб у git-історії було видно, що це не закриття виконанням.
- **При редагуванні (зміна пріоритету / статусу / категорії / назви):**
  оновити відповідні комірки рядка. Назва (текст посилання) може
  змінитись, але шлях `items/NNN-slug.md` — ні.
- **Перехід `відкрито` ↔ `🕒 відкладено`** змінює тільки колонку
  `Статус`. У відповідному item-файлі додається/видаляється поле
  `Активувати після:`.
- Колонки розділяти `|`, заголовок з `---` під ним — стандартний
  markdown-table.
- Назва завжди як markdown-посилання `[Текст](items/NNN-slug.md)`.

## Що НЕ робити з реєстром

- Не зберігати закриті/виконані борги (їх місце — git-історія).
- Не перенумеровувати інші рядки після видалення одного з них.
- Не змішувати у реєстрі рядки з різних репо чи дочірніх проєктів —
  кожен репо має свій `tech_debt/`.
- Не використовувати інші значення `Статус`, крім `відкрито` та
  `🕒 відкладено`.
- Не міняти набір колонок (`#`, `Назва`, `Категорія`, `Пріоритет`,
  `Статус`) — формат фіксований, щоб реєстр був прогнозовано читаним
  скілом.
